01 What is AtlasNexus?

AtlasNexus is the front-end and orchestration layer of the FIEngine platform — a fixed-income and multi-strategy systematic investment terminal. It turns a library of quant models covering curve calibration, factor (beta) allocation, relative-value (alpha) trading, futures strategies, multi-asset portfolio construction, and derivatives pricing into a unified, professional desk console.

Rather than requiring you to maintain a constellation of spreadsheets, standalone scripts, and separate risk systems, AtlasNexus integrates the full workflow — from raw market data to executed trade tickets — into a single, consistent environment driven by two browser-based consoles.

Why AtlasNexus?

One Source of Truth
Curves, signals, risk, and positions all read from the same set of pre-computed pipeline artifacts — no version drift across tools.
Systematic, Not Manual
Factor models, RV scanners, and curve fits run on a defined daily schedule. Decisions are informed by reproducible signals, not ad-hoc analysis.
End-to-End Coverage
From macro factor selection and risk sizing through to instrument-level trade tickets — every step of the workflow is handled in-platform.
Always Responsive
The UI displays pre-computed results rather than recalculating on every click. Even complex backtests and model outputs load instantly.
Full Auditability
Every EOD pipeline run writes a timestamped folder of JSON artifacts. You can replay and inspect any historical state of the desk.
CLI + UI Parity
Every action available in the console is also a command-line call — making it easy to automate, schedule, or script any part of the workflow.

02 Getting Started

Requirements

  • Python 3.9+ — conda environment dev (macOS/Linux) or prod (Windows).
  • Install dependencies: pip install -r requirements/production.txt
  • Optional — live data: WindPy (Wind terminal SDK) and TA-Lib. Without Wind the platform runs gracefully on cached/historical data.

Launching the Platform

macOS

./START_mac.command        # double-click in Finder, or run in Terminal

Windows

START_win.bat

Direct (any OS)

python main.py             # Daily Console → http://localhost:8080
python main.py daily-web   # Daily Console → http://localhost:8080
python main.py intraday-web # Intraday Console → http://localhost:8081

On launch the Daily Console opens automatically in your default browser. A small live log window appears on macOS and Windows showing startup progress. Suppress it with FI_SHOW_LOG_WINDOW=0 if preferred.

First-Run Checklist

  • Activate the conda environment and launch the Daily Console.
  • Open Run Center — confirm the latest EOD run status is green.
  • If data is stale, run Update Data, then Run EOD (see §6).
  • Browse the Market book — confirm curves and prices loaded correctly.
💡
Start here every morning.

The standard opening sequence is: Update Data → Run EOD → review the Summary book. Once that cycle is green, all five books are ready to use with today's calibrated state.

03 The Market Book

Market

The analyst's starting point — calibrated state of rates and credit markets, pricing tools, and directional context. Open this book first each morning to confirm data loaded and markets are well-behaved.

Sub-tabWhat it showsTypical use
Data Money-market rates, on-the-run & reference bonds, IRS forward rates with in-cell magnitude bars Sanity-check today's market levels
Pricer Bid/offer/mid for TBond, CBond, LBond, GBond, FR007 IRS, repo pairs, butterflies, futures basis Quote an instrument; read carry/roll greeks
Curves Calibrated yield and forward curves (TBond, CBond, IRS, credit) with fits and residuals Inspect curve shape, identify misfits
Surface Yield surface across tenor × maturity Spot rich/cheap pockets in the matrix
Trend Momentum and trend indicators, breakout signals Directional context for beta allocation

Reading the Pricer

The Pricer is the most frequently-used sub-tab. Key columns to understand:

  • Z-Score — the rich/cheap measure: standardised deviation of the current yield from its rolling mean. Positive = cheap; negative = rich.
  • Stationarity — whether the spread/yield series is mean-reverting (required for z-score signals to be actionable).
  • Carry (3M) / Roll (3M) / Total — return breakdown for holding the instrument three months. Total = Carry + Roll.
  • bp — all price changes are expressed in basis points.
ℹ
Filtering the Pricer

Use the dropdowns for asset type and term range (1–3Y, 3–5Y, 5–7Y, 7–10Y, 10–30Y) to narrow the table. Columns are sortable and filterable — sort by Z-Score descending to surface the richest opportunities immediately.

04 The Beta Book

Beta Book

Runs the systematic factor allocation workflow — from selecting which macro risk factors to take, to training the model that sizes them, through to the resulting portfolio and its bond/futures expression.

Sub-tabWhat it does
CandidatesFactor selection pool across IR / FX / EQ / CM. Pick the universe, then Train or Predict.
FactorFactor model detail — information coefficients (IC), feature weights, regime analysis.
BacktestWalk-forward backtest. Two sheets: Individual Factors (single-factor RFBT) and Portfolio (multi-asset).
PortfolioFactor risk-parity portfolio construction and current beta allocation.
Bond / FuturesInstrument-level expression of the beta book — how allocations translate into bonds and futures.

Running a Factor-Model Backtest

Navigate to Beta Book → Backtest → Individual Factors.

  1. Select a factor from the dropdown.
  2. Set the parameters below, then click ▶ Run Backtest & Save.
ParameterDefaultWhat it controls
Train window (months)12Rolling training window length
IC threshold0.05Minimum information coefficient to retain a feature
Top N features8Feature cap after IC-based selection
Backtest period (years)2Out-of-sample evaluation window

Results display as an IC table and a five-panel chart set: signal, predicted vs. realised return, equity curve, drawdown, and exposure. The trained model is persisted to input/models/factor_model_<YYYYMMDD>.joblib for daily signal generation.

⚠
Model training is a deliberate, periodic action

The daily EOD pipeline does not retrain the factor model — it only generates signals from the latest saved model. Run Backtest & Save here when you want to update the model with new data or changed parameters.

05 The Alpha Book

Alpha Book

The relative-value (RV) workspace — market-neutral spread, pairs, and volatility trades. Where the Beta Book takes directional macro risk, the Alpha Book targets mean-reverting mispricings that are structurally uncorrelated to those factors.

Sub-tabWhat it does
CandidatesRV scanner — z-score slider, signal clusters, rich/cheap ranking across the instrument universe.
SpreadSector PCA spreads, spread regression, treasury/policy/local/corporate, swap spreads, bond-swap spreads, futures term/net basis.
PairsPairs/spread trading: regression fit, hedge ratio, residual z-score, entry/exit signals.
VolatilityImplied vs. historical vol surfaces, term structure, skew/smile, vega exposure.
PortfolioAlpha book allocation and current RV positions.
BacktestHistorical RV strategy simulation, equity curves, drawdowns.

Reading a Pairs Signal

The Pairs tab fits a regression of one instrument on another and monitors the residual z-score against configurable entry/exit bands:

  • Entry signal: residual z-score crosses the entry band (e.g. ±1.5σ) — a candidate is flagged for the alpha book.
  • Exit signal: residual mean-reverts toward zero — the spread is unwound.
  • Confidence bands are derived from the residual standard deviation of the regression window.
📌
Always check stationarity first

A pairs trade is only statistically valid if the spread series is stationary (mean-reverting). The Pricer and Pairs tabs both surface a Stationarity status — only trade pairs where this is confirmed.

06 The Summary Book

The Summary book aggregates the Beta and Alpha books into a consolidated desk-level view. Use it at end-of-day to review combined exposure and generate trade tickets.

Sub-tabWhat it shows
Books Combined portfolio KPIs with beta/alpha allocation tables side by side.
Risk KPI strip (Total Long / Short / Net Exposure / Total DV01), net position by instrument, DV01 duration ladder (1Y–30Y), factor risk attribution, full position inventory.
Tickets Trade tickets generated from the delta between current target and actual positions.

The Risk sub-tab shows a DV01 ladder stacked by Bonds / Swaps / Futures, and factor risk attribution on a √-scale so that large rate-level factors do not visually dominate smaller commodity factors.

07 Run Center & Daily Operations

Run Center

The operator's control panel — drive the daily pipeline, monitor per-step status, and inspect logs. Most users interact with it once a day at the start of the session.

Step 2

→
Step 3

Check Summary book and per-step status in Run Center
→
Step 4
Analyse
Explore Market, Beta, and Alpha books for the day's opportunities

What the EOD Pipeline Produces

Each run writes a timestamped folder runs/<run_id>/ containing one JSON artifact per pipeline step:

runs/20260628-eod-083012/
├── run_meta.json           # manifest: mode, as-of date, per-step status
├── curves_result.json      # curve calibration
├── factors_result.json     # factor signal summary + backtest metrics
├── pairs_result.json       # pairs count and names
├── futures_result.json     # futures strategy metrics + equity curves
├── multiasset_result.json  # universe counts and names
├── derivatives_result.json # option greeks
└── factor_signals.json     # factor signal snapshot

Each pipeline step is isolated — if one step fails it is logged and skipped without aborting the rest. The web tabs read these artifacts directly.

09 Remote Access

server.bat starts a Cloudflare tunnel that exposes the local application publicly — useful for accessing the terminal from outside the office network or sharing a session with a colleague.

# Terminal 1 — start the app
python main.py daily-web

# Terminal 2 — proxy via cloudflared
server.bat
⚠
Security note

The Cloudflare tunnel makes the console publicly reachable. Use only on trusted networks and close the tunnel when the session ends. Do not expose the intraday console with live execution capabilities without additional authentication.

10 Troubleshooting

SymptomLikely causeFix
Curves/prices empty Data not retrieved — Wind unavailable or outside trading hours Run Update Data first, or confirm cached data is present
EOD step shows failed Single step error (isolated failure) Check Run Center logs; each step is independent — rerun only the failed step
Browser shows 0.0.0.0 unreachable Server binds all interfaces; browser rejects 0.0.0.0 Use http://localhost:8080 directly — the platform resolves a real host automatically
Backtest slow on Windows Multiprocessing fallback to serial mode Set FI_DISABLE_WINDOWS_CURVE_MP=1 to use serial mode explicitly
Log window not showing Platform default suppresses it Set FI_SHOW_LOG_WINDOW=1 and restart
Factor signals look stale Model not retrained after parameter change Re-run Beta Book → Backtest → Run Backtest & Save to update the saved model
Z-score not actionable Spread is non-stationary Check Stationarity column — only trade instruments where this status is confirmed

11 Glossary

EOD
End of day — the daily batch calibration and reporting run that produces all platform artifacts.
Beta
Systematic exposure to macro risk factors. The Beta Book manages this allocation.
Alpha
Market-neutral relative-value return from spread and pairs trading. The Alpha Book manages this.
DV01
Dollar value of 1 basis point — the interest-rate sensitivity of a position or portfolio expressed per bp move.
IC
Information coefficient — rank correlation between a predictive feature and forward realised return. Higher is better.
ICIR
IC information ratio — IC mean divided by IC standard deviation. A signal-quality measure analogous to Sharpe.
Carry / Roll
Carry: return from holding a bond at its yield (coupon minus funding). Roll: return from the bond's yield moving down the curve over time. Total = Carry + Roll.
Z-score
Standardised deviation of a current value from its rolling mean, expressed in standard deviations. The primary rich/cheap gauge.
Stationarity
Statistical property of a time series that mean-reverts — a prerequisite for z-score signals to be actionable as trade candidates.
RV
Relative value — market-neutral trading of spread or pairs relationships, targeting mean-reversion of a mispricing.
CTD
Cheapest-to-deliver — the bond most economically delivered into a futures contract given current prices and conversion factors.
IRR
Implied repo rate — the breakeven financing rate embedded in a futures basis trade.
Net basis
Futures basis net of carry (coupon income minus financing cost) — the cleaner measure of futures richness or cheapness.
RFBT
Risk Factor BackTest — the single-factor out-of-sample backtest run from Beta Book → Backtest → Individual Factors.
bp
Basis point — 0.01% or 0.0001 in yield terms. The standard unit for expressing yield changes and price moves in fixed income.
